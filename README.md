**Movie Notebook API** — это RESTful бэкенд-сервис для социальной платформы киноманов. 

Проект спроектирован как гибрид базы данных фильмов и социальной сети. На данный момент реализовано ядро: интеграция со сторонними кино-API с "умным" локальным кэшированием, ролевая система, рецензии и рейтинги. В будущем проект эволюционирует в платформу для создания контента (статей) и обмена фильмами с друзьями.

## 🚀 Текущий функционал (Реализовано)

* **Умный поиск и кэширование (Kinopoisk API):** Интеграция с внешним API с использованием современного `RestClient`. Система использует сложную кастомную пагинацию (`PaginationCalculator`) и кэширует результаты в локальную БД (PostgreSQL), чтобы обходить лимиты внешнего сервиса и снижать время ответа.
* **Аутентификация и безопасность:** JWT-токены, хэширование паролей (BCrypt), разделение прав доступа (`ROLE_USER`, `ROLE_ADMIN`). Учет 152-ФЗ (согласие с политикой).
* **Система рейтингов и рецензий:** 
  * Оценки от 1 до 10 с автоматическим пересчетом среднего балла фильма (в реальном времени).
  * Написание рецензий с реализацией паттерна **Soft Delete** (мягкое удаление без потери связей в БД).
* **Модерация контента:** Система жалоб (Reports) на пользовательские рецензии. Разрешение конфликтов администраторами (удаление отзыва или отклонение жалобы).
* **Архитектура:** Использование `MapStruct` для маппинга DTO, `Caffeine` для in-memory кэширования, `Flyway` для миграций БД.

## 🗺️ Roadmap: Видение и планы на будущее

Проект активно развивается. Запланирован переход от простой системы отзывов к полноценному community-driven сервису:

- [ ] **Статьи и лонгриды:** Возможность для пользователей писать полноценные блоги/статьи о кино.
- [ ] **Интерактивные виджеты фильмов:** Интеграция мини-карточек фильмов прямо в текст статей. Читатель сможет в один клик добавить фильм из статьи в свой список "Хочу посмотреть".
- [ ] **Кастомные коллекции (Списки):** Создание пользовательских вкладок/групп фильмов (например: "Планы на выходные", "Любимый артхаус", "Смотреть с девушкой").
- [ ] **Социальный граф (Друзья):** Система добавления в друзья.
- [ ] **Шеринг и Лента активности:** Возможность поделиться фильмом или целой подборкой с друзьями. Формирование умной ленты новостей на основе активности друзей (кто что посмотрел, какую оценку поставил, что добавил в "Буду смотреть").

## 🛠 Технологический стек

* **Язык:** Java 21
* **Фреймворк:** Spring Boot (Web, Data JPA, Security, Cache)
* **База данных:** PostgreSQL 16 + Flyway (Миграции)
* **Безопасность:** Spring Security + JWT
* **Интеграции:** Spring RestClient
* **Утилиты:** MapStruct (маппинг), Lombok, Caffeine (кэш)
* **Инфраструктура:** Docker, Docker Compose

## ⚙️ Технические особенности проекта

* **Оптимизация внешних запросов:** Реализован алгоритм проверки "свежести" данных о фильме (`movieStaleThreshold`). Если фильм есть в базе, но данные устарели — бэкенд незаметно для пользователя обновит их.
* **Продвинутая пагинация:** Написан собственный `PaginationCalculator`, который синхронизирует внешнюю пагинацию (по 20 элементов) со внутренними запросами клиента любого размера (например, по 15 элементов на страницу).
* **Изоляция слоев:** Строгое разделение Entity и DTO, обработка исключений через `GlobalExceptionHandler`.

## 📦 Запуск проекта локально

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/your-username/movie-notebook-api.git
   ```
2. Настройте переменные окружения (токен Кинопоиска, секреты JWT) в `application.yml` или `.env` файле.
3. Запустите инфраструктуру (БД) через Docker:
   ```bash
   docker-compose up -d
   ```
4. Запустите приложение:
   ```bash
   ./mvnw spring-boot:run
   ```
```
